# Support

## Where to ask for help

- Usage questions and troubleshooting: GitHub Issues
- Bug reports: GitHub Issues (include reproduction steps)
- Security issues: follow `SECURITY.md` (private disclosure)

## Before opening an issue

- Read `README.md`, `CONTRIBUTING.md`, and relevant files under `docs/references/`.
- Search for existing issues and closed issues.
- Reproduce on the latest version when possible.

## What to include in support requests

- OS and shell
- Node and `pnpm` versions
- Exact command run and full error output
- Minimal reproduction repo or steps
- Expected behavior vs actual behavior

## Issue triage expectations

- Initial maintainer response target: within 5 business days
- Severity-based handling:
  - Critical/security: prioritized immediately
  - Regressions and release blockers: high priority
  - Feature requests: reviewed and prioritized during roadmap planning

## Scope and SLA

This is an open-source project maintained on a best-effort basis.
No guaranteed response or resolution SLA is provided.
