# harness-kit ‚Äî Project Map

> Pointer map for AI agents. For implementation details, follow file links.

## What This Project Does

CLI that scaffolds AI agent harness environments. `harness-kit init` ‚Üí wizard ‚Üí copies `.claude/`, `CLAUDE.md`, `AGENTS.md`, `harness.json`, MCP config, rules, and skills into the user's project.

## Bundle Registry

Bundles are TypeScript files in `src/registry/bundles/{workflow,stack,techstack}/<name>/manifest.ts`. Each exports a `BundleManifest` object with:
- `common.artifacts` ‚Äî installed for all roles (usually contains the MCP entry)
- `roles` ‚Äî keyed by `BundleCategory`; each role entry can mark `recommended: true`
- `defaultRole` ‚Äî the role used when this bundle is selected

**Bundle categories (9, see `packages/core/src/types.ts`):** `git-workflow`, `workflow-preset`, `memory`, `browser`, `search`, `scrape`, `mcp-tool`, `stack`, `techstack`

- `stack` ‚Äî language-base bundles (bucket B): forbidden from containing `type:'stack'` artifacts (cycle prevention)
- `techstack` ‚Äî tool/framework bundles (bucket A): may inherit a stack via `{ type: 'stack', ref: '<lang>' }` artifact

Adding a new option = add a bundle manifest, no wizard code change.

## Wizard Flow

```
projectInfo ‚Üí techStackSelect ‚Üí detectTooling ‚Üí harnessConfig ‚Üí previewApply
```

- `harnessConfig` ‚Äî multiselect per zone, all options from registry (`getBundlesByCategory`)
- `previewApply` ‚Äî renders templates, writes files via Listr2
- All default selections from `getRecommendedByCategory`

## Project Harness

This repo scaffolds itself with its own CLI. Artifacts under `.agents/`, `.claude/`, and `.githooks/` are generated by `harness-kit init` from the registry ‚Äî edit the upstream source in `packages/harness-kit/{skills,rules,agents,git-hooks}/`, not the installed copy.

```
.agents/skills/         ‚Üí universal skill copies (for Cursor, Codex, Gemini CLI, etc.)
.claude/
  skills/               ‚Üí symlinks into .agents/skills/*
  rules/                ‚Üí always-loaded router rules (12 files)
  agents/               ‚Üí sub-agent definitions (code-reviewer, security-reviewer, skill-evaluator)
  hooks/                ‚Üí Stop / PreToolUse / PostToolUse shell hooks
  memory/               ‚Üí committed project + reference memories (see memory skill)
    INDEX.md            ‚Üí human-maintained TOC
    project.md          ‚Üí hot cache (pointers only, <100 lines)
    project/            ‚Üí deep store (project-state, decisions, gotchas)
.githooks/              ‚Üí dispatcher + per-bundle .d/ scripts (commit-msg, pre-commit)
examples/               ‚Üí manual test projects (not in pnpm workspace)
  nextjs-app/           ‚Üí sample Next.js project used for smoke testing
```
